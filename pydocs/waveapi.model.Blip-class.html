Models a single blip instance.

Blips are essentially elements of conversation. Blips can live in a hierarchy of blips. A root blip has no parent blip id, but all blips have the ids of the wave and wavelet that they are associated with.

Blips also contain annotations, content and elements, which are accessed via the Document object.
