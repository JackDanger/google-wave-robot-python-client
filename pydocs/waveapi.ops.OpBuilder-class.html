type OpBuilder

source code

object --+
         |
        OpBuilder

Wraps all currently supportable operations as functions.

The operation builder wraps single operations as functions and generates operations in-order on its context. This should only be used when the context is not available on a specific entity. For example, to modify a blip that does not exist in the current context, you might specify the wave, wavelet and blip id to generate an operation.

Any calls to this will not reflect the local context state in any way. For example, calling WaveletAppendBlip will not result in a new blip being added to the local context, only an operation to be applied on the server.
